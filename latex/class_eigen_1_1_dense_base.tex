class Eigen\+::\+Dense\+Base$<$ Derived $>$}
Base class for all dense matrices, vectors, and arrays. 

This class is the base that is inherited by all dense objects (matrix, vector, arrays, and related expression types). The common Eigen API for dense objects is contained in this class.

Indeed, {\ttfamily is\+Approx(zero)} returns false unless {\ttfamily \texorpdfstring{$\ast$}{*}this} itself is exactly the zero matrix or vector. This is the "{}in place"{} version of \doxylink{class_eigen_1_1_dense_base_ac8952c19644a4ac7e41bea45c19b909c}{transpose()}\+: it replaces {\ttfamily \texorpdfstring{$\ast$}{*}this} by its own transpose. Thus, doing 
\begin{DoxyCode}{0}
\DoxyCodeLine{m.transposeInPlace();}

\end{DoxyCode}
 has the same effect on m as doing 
\begin{DoxyCode}{0}
\DoxyCodeLine{m\ =\ m.transpose().eval();}

\end{DoxyCode}
 and is faster and also safer because in the latter line of code, forgetting the \doxylink{class_eigen_1_1_dense_base_a5df64c66228ba75bbc66db2584185527}{eval()} results in a bug caused by aliasing.

Notice however that this method is only useful if you want to replace a matrix by its own transpose. If you just need the transpose of a matrix, use \doxylink{class_eigen_1_1_dense_base_ac8952c19644a4ac7e41bea45c19b909c}{transpose()}.

\begin{DoxyNote}{Note}
if the matrix is not square, then {\ttfamily \texorpdfstring{$\ast$}{*}this} must be a resizable matrix. This excludes (non-\/square) fixed-\/size matrices, block-\/expressions and maps.
\end{DoxyNote}
